# Synthetic Data Pack

This directory contains deterministic sample datasets created exclusively for portfolio use. Every file is fictitious, safe to share publicly, and centered on a synthetic version of the Maker Info service operations system.

## Included Files

- `dim_company.csv`: client account dimension with retail and B2B organizations
- `dim_customer.csv`: customer contacts linked to retail and B2B accounts
- `dim_technician.csv`: technician roster with squads and specialties
- `fact_service_order.csv`: order-level operational and commercial data for the system
- `fact_order_status_history.csv`: timestamped status transitions for SLA and backlog analysis
- `fact_cost_entry.csv`: linked and unlinked cost records for profitability analysis
- `fact_customer_contact.csv`: recall, quote follow-up, and account review interactions
- `service_orders_raw.csv`: intentionally messy extract designed for the Python cleaning example
- `service_orders_clean.csv`: cleaned export generated by the pandas transformation script

## How The Data Was Built

- The generator script lives in `Python_Automation/generate_synthetic_portfolio_data.py`
- Output is deterministic, which means the same seed always recreates the same files
- Dates, names, accounts, and operational events were designed to look realistic without matching any real entity

## Suggested Usage

- Load the service order tables into PostgreSQL, DuckDB, or Databricks to test the SQL challenges
- Use `service_orders_raw.csv` with `Python_Automation/clean_service_orders_data.py`
- Use `fact_service_order.csv`, `fact_cost_entry.csv`, and `fact_customer_contact.csv` as dashboard prototyping sources
- Install dependencies from `requirements.txt` before running the pandas cleaning workflow
